SESSION 1 — StockSense Project Start
=====================================

## What we're building

A C# app with two projects:
- StockSense.Core   — all the logic (no user interface)
- StockSense.Console — the menu the user sees

These are two separate compiled files. That alone earns 1 point from the grader.

## Files written this session

### SignalType.cs
Not a class — it's an enum. Enums are just a named list of options.
This one lists the signal types the app can produce:

    None, Buy, Sell, Strong, Weak

The [Flags] part means you can combine them: Buy | Strong = StrongBuy.
That's the bitwise operations requirement. (1 pt)

---

### StockPrice.cs
Holds one day of stock data:

    Date:   2024-03-15
    Open:   181.00
    High:   183.50
    Low:    180.20
    Close:  182.34
    Volume: 55,123,400

On top of that, 5 grading requirements were added to it:

- IComparable  — so you can sort a list of prices by date (oldest first)
- IEquatable   — so two prices with the same date are considered equal
- IFormattable — so you can print it in different ways ("S" short, "L" long)
- Operators    — so you can write:  if (price1 < price2)  instead of the ugly CompareTo version
- Deconstructor — so you can write:  var (date, close) = price;  as a shortcut

---

### IndicatorPoint.cs
Holds one calculated indicator value:

    Date:  2024-03-15
    Value: 72.4   ← this would be an RSI reading for that day

Nothing fancy. Just a container for "on this date, the indicator was this value."
Adds: IEquatable, Deconstructor.

---

### MacdPoint.cs
Same idea as IndicatorPoint, but MACD produces 3 values at once:

    Date:      2024-03-15
    Macd:      1.23
    Signal:    0.98
    Histogram: +0.25

Adds: IFormattable, Deconstructor.

---

## Why models first?

Because every other file depends on them. You can't write the RSI calculator
without IndicatorPoint to put results into. You can't write the API service
without StockPrice to store the data. Models first, then logic, then the UI.
